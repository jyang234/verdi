# lines ignored. <layer> is a 1-based integer; <path> is relative to this

# rate-lock/rate-lock-v2 (the OTHER supersession-chain fixture originally

# `supersession:` block, and VL-015 (internal/lint/vl015.go) reads a
# predecessor's object manifest via `git show <pred.frozen.commit>:
# <pred.RelPath>`, which only succeeds if the predecessor already exists,
# with matching content, AT that exact commit — impossible for a file
# introduced fresh in this shared layer, since every citable earlier
# commit here predates rate-lock's own existence. The dedicated history
# solves it with a draft-then-frozen two-commit sub-sequence, exactly as
# loan-workflow's already did; see corpus_test.go's goldenHeadsV2 comment
# and internal/lint/v2clean_test.go's own doc comment for the rest of the
# mechanism. escrow-notify/escrow-notify-v2 did NOT need this: 02 §Object

# (artifact.SpecFrontmatter.Validate), so that STORY-rung pair is fully
# expressed by `supersedes` plus the terminal status flip alone (03 §rung
# 3) and never trips VL-015 — it stays a normal layers.txt-tracked pair,
# both halves in this one shared commit.
#
# borrower-update-mobile/deviation-report.md is deliberately NOT listed
# here even though it also pins layer 3's head: its sibling spec.md is
# itself not layers.txt-tracked (it is part of the pre-existing "v2
# fixture overlay" set — examples/showcase/.verdi/ content beyond these
# layers, folded by internal/dex's own buildDexFixtureRepo and decoded
# directly by internal/artifact/v2fixture_test.go). Listing only the
# deviation report here would orphan it inside the narrower
# layers.txt-only builds (internal/corpus, internal/index,
# internal/workbench): those build fixturegit repos strictly from this
# file, so a specs/active/borrower-update-mobile/ directory would appear
# with a deviation-report.md but no spec.md — internal/storyresolve's
# matchStoryRef reads every specs/active/*/ directory's spec.md
# unconditionally and errors on the missing file (proven by running the
# suite before this note was added). Keeping the deviation report as a
# plain, non-layered file (like its spec.md sibling) mirrors the existing
# discipline instead of inventing a new one, and is still fully covered by
# corpus_test.go's whole-tree pin-name walk.
#
# public-rollout-plan Task 1.4 extends the same exception to the five
# obligation files authored for borrower-update-api and

# `verifies` edge to its story's spec.md, and VL-019 refuses any obligation
# whose verifies target does not resolve to a real story spec in the same

# story's spec.md is layers.txt-tracked, layers.txt-tracking only the
# obligation would orphan it exactly as a layered deviation-report.md would
# have been orphaned above — same root cause, same fix. These five files
# are instead wired into internal/lint/v2clean_test.go's
# buildV2FixtureCorpusRepo layerC, alongside the two story spec.md files
# they verify, so TestV2FixtureCorpus_LintsClean proves them genuinely
# lint-clean (not merely present) in a real git history that also contains
# their verifies targets. They remain fully covered by corpus_test.go's
# whole-tree pin-name walk, same as the deviation report above.
