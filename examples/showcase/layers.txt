# layers.txt — maps corpus files to fixturegit commit layers (PLAN.md §4:
# "a layers.txt (or similar) manifest declaring which files belong to
# which fixturegit commit layer").
#
# Format: one "<layer> <path>" pair per line, blank lines and "#" comment
# lines ignored. <layer> is a 1-based integer; <path> is relative to this
# directory (testdata/corpus/) and is also the path written into the
# fixturegit-built repo's working tree.
#
# Only committed-zone files (.verdi/, minus data/) appear here. Files
# under mutable/ and derived/ model the store's mutable and derived zones,
# which VL-013 forbids git-tracking in the real store — corpus_test.go
# decodes them directly from testdata/corpus/ on disk instead of routing
# them through fixturegit.
