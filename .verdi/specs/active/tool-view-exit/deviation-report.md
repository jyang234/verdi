## Judged

- **judged-escape-during-inline-rename-also-exits** [UNDISPOSITIONED]: boarddiagram.js's page-level Escape handler claims to stand down while the inline rename editor is open, but it fires in the bubble phase after the rename input's own target-phase keydown handler has already run abandon() and removed the input from the DOM (startInlineRename neither calls stopPropagation nor is checked before removal). The subsequent document-level check `document.querySelector('.diagram-rename-input')` therefore finds nothing and navigates. One Escape press meant to cancel a rename both abandons the rename AND exits the tool view — the exact in-progress-gesture discard the implementation's own comment (assets/boarddiagram.js ~line 595) says it prevents, and contrary to dc-1's own framing that in-editor overlays close in place without navigating away. No e2e covers Escape while a rename is open, so ac-1's proven paths never exercise this. (confidence 0.70)
- **judged-branch-board-entry-mislabeled-as-known-origin** [UNDISPOSITIONED]: Per-branch boards (/b/{branch}/board/spec/{name}) dispatch into boardSpecServer instances that share loadBoard, which now calls attachDiagramEditorHrefs(proj, s.root, name) unconditionally (boardspec.go:239) — so a branch board's pinned diagram reference card also renders an editor link carrying board=<name>. An operator entering the editor from a branch board gets an exit affordance labeled 'back to board: <name>' (Known=true, since <name> resolves in the serving root's specs/active) that navigates to the UNPREFIXED /board/spec/<name> — the serving checkout's board, not the branch board the session actually came from. dc-1's inventory claim ('never from a per-branch board') is true only of the editor's route, not of the entry link the branch board demonstrably renders; unlike the corpus-page gap dc-3 discloses, this case affirmatively asserts a known origin that is wrong, undisclosed — a sharper breach of ac-1's 'that exact spec board' and dc-3's honesty bar than the disclosed narrowing. (confidence 0.55)
- **judged-exit-races-pending-autosave** [UNDISPOSITIONED]: Escape (window.location.href = state.exitHref) and the exit affordance navigate immediately without flushing the editor's 400ms debounced autosave or awaiting the in-flight savingChain (boarddiagram.js saveNow/saveTimer). An operator who types in the code pane and exits within the debounce window silently loses those bytes: the originating board is confirmed restored per ac-1, but the story's new exit paths introduce a silent-discard channel that rubs against board-editor ac-3's every-save-stores-your-bytes posture and the parent's never-a-silent-omission stance. No test covers exiting with a dirty, not-yet-saved pane. (confidence 0.35)
