# The partition is the finding's own declaration, not a wall allowlist

The static evidence must show the classification seam on lint.Finding (the
optional wall-locus declaration of dc-3 — object anchor or spec-level
marker) populated by the raising rules themselves, and the wall's routing
reading ONLY that seam plus the finding's Path: no switch or map over VL
rule ids inside internal/workbench decides card vs case file vs off-wall.
It must also show the exclusion is fail-closed: a finding that declares no
locus never reaches the projection even when its Path lies inside the
spec's own directory — plumbing rules and decode paths simply declare
nothing, and no enumerated rule-id list exists anywhere in the wall.
