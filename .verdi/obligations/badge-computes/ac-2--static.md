# The partition is the finding's own declaration, not a wall allowlist

The static evidence must show the classification seam on lint.Finding (the
optional object-anchor of dc-3) populated by the raising rules themselves,
and the wall's routing reading ONLY that seam plus the finding's Path: no
switch or map over VL rule ids inside internal/workbench decides card vs
case file vs off-wall. It must also show decode failures and repo-wide
loci excluded structurally (they carry no in-spec path or anchor), not by
an enumerated rule-id list.
